# AGENT RULES (для любых моделей)

Этот файл обязателен для всех агентных сессий (GPT/Codex, Claude, Gemini и др.).

## 1) Перед началом работы

1. Прочитай `ai-context/SESSION_CONTEXT.md`.
2. Быстро проверь актуальное состояние репозитория (`git status`, ключевые файлы задачи).

## 2) Во время работы

1. Не ломай существующий flow без явной причины.
2. Если меняешь архитектуру/контракты API — зафиксируй это в контексте.
3. Если есть риск/компромисс — явно опиши его в `WORKLOG.md`.

## 3) После выполнения (обязательно)

1. Обнови `ai-context/WORKLOG.md` (что сделано, какие файлы, результат).
2. Обнови `ai-context/SESSION_CONTEXT.md` (текущее состояние проекта).

## 4) Формат записи в WORKLOG

- Дата/время
- Модель/сессия
- Цель
- Изменения (по файлам)
- Деплой/проверка
- Риски/заметки

## 5) Запреты

- Не удалять файлы из `ai-context/` без явной просьбы владельца.
- Не оставлять задачу «в подвешенном состоянии» без заметки в `WORKLOG.md`.
- Не выполнять destructive git-команды (`reset --hard`, `checkout --`, force push) без явного запроса владельца.
