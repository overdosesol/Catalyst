# Code Of Conduct

Catalyst should be a useful and respectful place for maintainers, users and
contributors.

## Expected Behavior

- Be direct, kind and specific.
- Assume good intent, but verify technical claims with evidence.
- Keep feedback focused on the code, docs, behavior and security impact.
- Respect that maintainers may decline changes that do not fit the project
  scope.

## Unacceptable Behavior

- Harassment, insults, threats or discriminatory language.
- Sharing private information, secrets, logs or production details without
  permission.
- Filing public vulnerability details instead of following `SECURITY.md`.
- Repeatedly pushing a rejected change without new technical evidence.

## Enforcement

The maintainer may edit, hide or remove issues, comments and pull requests that
violate this code of conduct. Serious or repeated violations may result in being
blocked from the repository.
